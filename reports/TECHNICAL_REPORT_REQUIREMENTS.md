# Требования к техническому HTML-отчёту

Этот документ фиксирует обязательную структуру для `reports/partner_weekly_report_technical*.html`.

## Обязательные блоки

Технический отчёт должен содержать:
- краткий заголовок и дату снимка;
- инкрементальный weekly snapshot по git/GitHub;
- секцию проверок и ограничений текущего прогона;
- отдельную секцию `Этапы существенных изменений с момента создания репозитория`;
- инженерное резюме.

## Инкрементальная weekly-часть

Weekly snapshot должен описывать только новые изменения относительно предыдущего технического отчёта.

Это значит:
- новые коммиты за период;
- новые закрытые issue / merged PR / выпущенные артефакты за период;
- новые риски и новые локальные изменения текущего снимка.

Weekly snapshot не должен каждый раз заново пересказывать всю историю репозитория.

## Секция этапов

Секция этапов обязательна для всех будущих генераций.

Требования:
- этапы должны быть визуально выделены как progression/timeline;
- каждый этап должен иметь номер, короткий заголовок и статус;
- для каждого этапа нужно кратко показать:
  - ключевые изменения;
  - инженерный эффект;
  - связанные issue, ветки или артефакты, если они доступны;
- этапы должны описывать не только последний commit, а накопленную картину существенных изменений с момента создания репозитория;
- первый этап должен объяснять исходную точку репозитория: когда он был создан и какой зрелый baseline уже содержал.
- секция этапов должна быть append-only:
  - существующие этапы не пересчитываются каждую неделю заново;
  - новый этап добавляется только если появился действительно новый существенный рубеж проекта.

## Язык

Весь человекочитаемый текст отчёта остаётся на русском языке.

Без перевода допустимо оставлять только точные технические идентификаторы:
- commit hashes;
- имена веток;
- issue/PR номера;
- точные имена файлов, путей, классов и команд.
