# Политика языка для партнёрских графических отчётов

Все будущие отчёты в `reports/partner_weekly_report*.html` должны использовать русский язык во всем человекочитаемом тексте.

Обязательные зоны локализации:
- заголовки страниц и бейджи;
- подзаголовки, описания, футеры;
- подписи метрик, карточек, таймлайнов, секций и рисков;
- redirect-страницы `*_latest.html`;
- легенды, статусы, поясняющие подписи и служебный текст.

Допустимо оставлять без перевода только точные технические идентификаторы и артефакты:
- имена файлов и путей;
- commit hashes;
- названия классов/API/пакетов и команд, если они приводятся как точные токены, например `BotPanel`, `ONNX Runtime`, `verify-dotnet`, `Market/Servers`.

Правило для будущих генераций:
- сначала формировать весь narrative/UI-текст на русском;
- затем вставлять точные технические токены без перевода только там, где это нужно для однозначности.

Дополнительное правило для технической версии отчёта (`partner_weekly_report_technical*.html`):
- weekly-часть technical report должна быть инкрементальной и описывать только новые изменения относительно предыдущего отчёта;
- технический отчёт должен содержать отдельную визуально выделенную секцию с этапами существенных изменений с момента создания репозитория;
- этапы должны читаться как progression/timeline, а не как обычный абзац;
- для каждого этапа нужно кратко фиксировать:
  - что именно изменилось;
  - зачем это важно для инженерного состояния репозитория;
  - на какие крупные issue/артефакты это опирается, если они есть.
- timeline этапов должен быть накопительным append-only блоком: не пересказывать всю историю заново в weekly narrative, а только дополняться при появлении нового существенного этапа.
